# Core Beliefs — 설계 핵심 원칙

## 1. 에이전트 우선 (Agent-First)

복잡한 분석 작업은 전문 에이전트에 위임한다. 단일 모놀리식 로직보다 역할이 명확한 에이전트 파이프라인이 유지보수와 확장에 유리하다.

## 2. 데이터와 판단의 분리

- `src/data/` — 외부 데이터 수집 (side-effect 허용)
- `src/analysis/` — 순수 함수 기반 분석 (no side-effect)
- `src/agents/` — 위 두 레이어를 조합한 판단 로직

## 3. Slack은 유일한 출력 채널

분석 결과는 `output/`에 임시 저장 후 Slack으로 전달한다. UI 없이 알림만으로 충분하다는 전제.

## 4. 레짐 기반 포지션 사이징

시장 국면(BULL/BEAR/VOLATILE/SIDEWAYS)을 먼저 확인하고, 신뢰도(%)에 따라 배팅 크기를 결정한다. 무조건적인 풀 포지션 진입 금지.

## 5. 환경변수로 모든 민감 정보 관리

API 키, Webhook URL 등은 코드에 절대 하드코딩하지 않는다.
